# PitStop-Linux → macOS v0.3.1 Feature Catch-Up — Design

**Date:** 2026-07-01
**Status:** Approved design, ready for implementation planning
**Component:** PitStop for Linux (Rust, `ksni` system-tray app)
**Upstream:** ports macOS PitStop v0.2.0 → v0.3.1 (`../pitstop`, Swift/AppKit)

## Problem

The Linux port (`pitstop-linux`) was forked from macOS PitStop at roughly v0.1.
Since then the Mac app shipped v0.2.0 → v0.3.1, adding four feature clusters the
Linux port lacks. This design ports them faithfully, adapting each to Linux
where the platform differs (tray-only UI, file/keyring credential storage,
`xdg-open`/`zenity`, source-based install).

## Feature delta being ported

| # | Feature | Mac version | Effort |
|---|---------|-------------|--------|
| 1 | Least-squares usage projection (per-window) | v0.2.1 | S |
| 2 | Usage-dashboard links (↗ per provider section) | v0.3.1 | S |
| 3 | In-app OAuth re-login (Claude + Codex) | v0.3.0 | M |
| 4 | Google Gemini provider — **Antigravity** surface | v0.3.0 | L |
| 5 | Updater: GitHub-release check + rebuild & relaunch | v0.2.0 | M |

## Scope decisions (this environment)

Grounded in what is actually installed on the target machine:

- **Claude Code** — active (`~/.claude/.credentials.json`). Testable.
- **Codex CLI** — installed and logged in (`~/.codex/auth.json`). Testable.
- **Gemini = Antigravity** (CLI + 2.0 app) — the Gemini *CLI* and Gemini *app*
  are **not** present on Linux. Antigravity's OAuth token lives in the **GNOME
  keyring** (Secret Service, go-keyring item `service=gemini, account=antigravity`),
  **not** a plaintext file, and `~/.gemini/google_accounts.json` (a Gemini-CLI
  artifact) is absent.

Decisions made during brainstorming:

1. **Gemini/Antigravity: usage + account switching.** Read the token from the
   GNOME keyring, show Code Assist usage, and support one-click switching by
   writing the token back into the keyring. The Antigravity **ToS discourages
   token rotation** — surface that caveat in the switch notification and README;
   keep auto-switch opt-in (already the default).
2. **Updater: full rebuild & relaunch.** `git pull` + `cargo build --release` +
   reinstall + re-exec, since the Linux app is distributed as source.
3. **Also include** (all low-risk): OAuth re-login (Claude + Codex), dashboard
   links, least-squares projection.

## Non-goals

- **Gemini CLI file surface** (`~/.gemini/oauth_creds.json`) and
  `google_accounts.json` handling — not installed here. The provider is
  Antigravity-only on Linux (the mirror of the Mac decision, which was
  CLI-primary). The code should be structured so a CLI surface *could* be added
  later, but v1 does not implement it.
- **Claude Desktop** — already dropped in the Linux port (no Linux build);
  unchanged.
- Re-login for the **live/active** account of any provider (the CLI keeps it
  fresh; re-login is inactive-rows-only, preserving the "never touch live"
  invariant).
- A general `Provider` trait refactor — three hand-coded providers is fine and
  matches both the current Linux code and the Mac app.

## Current Linux baseline (starting point)

- **Runtime:** single `tokio` task; `tokio::select!` loop over an mpsc
  `Action` channel + a 120 s refresh timer (`app.rs` `Engine::run`). No threads,
  no locks. Tray clicks arrive as `Action`s.
- **Tray:** `ksni` `Tray` impl; menu rebuilt every render from a `TrayView`
  snapshot (`tray.rs`). Section headers are clickable `StandardItem`s; detail
  lines are `disabled()`; settings are a `SubMenu` of `CheckmarkItem`s.
- **Providers:** hand-coded `Claude` + `Codex` (`model.rs` `Provider` enum).
  Codex is the direct template for a third provider (`codex.rs` + `codex_store.rs`).
- **Credentials:** file-based, `0600`. Live stores at `~/.claude/.credentials.json`
  and `~/.codex/auth.json`; snapshots under `~/.config/pitstop/accounts/`;
  non-secret metadata in `~/.config/pitstop/{profiles.json,codex-profiles.json,settings.json}`.
- **Account state (implicit):** `active_email` / `codex_live_email`,
  `store.profiles`, `needs_action: HashSet` (set on `ApiError::Unauthorized`),
  `fetch_error: map`. **No Login affordance yet.**
- **Projection:** `projected_full()` — first-vs-last slope, ≥3 samples, ≥300 s,
  blended per-account (not per-window).
- **Deps present:** `tokio` (full), `reqwest` (json), `serde`/`serde_json`,
  `ksni`, `base64`, `chrono`, `ab_glyph`, `png`, `anyhow`. No keyring, sha2,
  rand, or dialog crate.
- **No updater / version display.**

## Data-model changes (`model.rs`)

- `Provider` gains `Gemini`; update `Provider::ALL`, `title()` (`"Gemini"`), and
  add `dashboard_url()` for all three:
  - Claude → `https://claude.ai/new#settings/usage`
  - Codex → `https://chatgpt.com/codex/cloud/settings/analytics#usage`
  - Gemini → `https://gemini.google.com/usage`
- Account key stays provider-namespaced: Gemini key = `"gemini:<email>"`.
- Surface tag for Gemini rows = `"Antigravity"` (single surface on Linux).
- Row model gains a **login affordance**: when a row is in `needs_action`,
  inactive, and switchable, the tray renders a coral **Login** action in place
  of the Switch action.

## New modules

| Module | Purpose |
|---|---|
| `oauth.rs` | PKCE (S256 verifier/challenge/state), authorize-URL builder, token exchange, `LoginCoordinator` (browser-open → await code → exchange → identity check → persist), and the `ClaudeLoginAdapter` / `CodexLoginAdapter` / `GeminiLoginAdapter` implementations behind a small `LoginAdapter` trait. |
| `loopback.rs` | One-shot HTTP callback server on `127.0.0.1` (`tokio::net::TcpListener`), parses the first `GET /<path>?code=…&state=…`, returns a "you can close this tab" page, yields `(code, state)`; configurable/fixed port; ~3 min timeout; cancellable. Replaces macOS `NWListener`. |
| `secret_service.rs` | go-keyring-compatible Secret Service client (via the `secret-service` crate): `get(service, account) -> Option<String>` and `set(service, account, value)` matching zalando/go-keyring's schema (attributes `service` + `username`, label `"<service>/<account>"`, value carries the `go-keyring-base64:` prefix). Used only by `gemini_store`. |
| `gemini.rs` | Antigravity credential-blob parse/patch, Google token refresh (refresh-token grant against the Antigravity client), `loadCodeAssist` + `retrieveUserQuota` calls, usage-window mapping + binding/extras, model-name shortening, and `userinfo` email lookup. |
| `gemini_store.rs` | Live keyring read/write, per-account snapshots to `~/.config/pitstop/accounts/`, `capture_current`, `switch_to`, `blob(for, is_active)`, active-email tracking. Analogous to `codex_store.rs` but keyring- rather than file-backed. |
| `updater.rs` | GitHub-releases poll, semver compare, `UpdateInfo { version, url, can_rebuild }`, and `rebuild_and_relaunch()` (locate checkout → `git pull` → `cargo build --release` → reinstall → re-exec). |

## New dependencies (`Cargo.toml`)

- `sha2` — PKCE S256 challenge (`base64url(SHA256(verifier))`).
- `rand` — PKCE verifier + `state` randomness.
- `secret-service` — GNOME keyring read/write for Antigravity (async, tokio
  runtime feature).
- Authorize-URL query assembly + callback parsing use **`reqwest::Url`**
  (already available via `reqwest`) — no new `url` crate.
- No dialog/browser/clipboard crate — shell out to `xdg-open`, `zenity`,
  `xclip` via `std::process::Command`.
- Semver compare: hand-rolled `(u64,u64,u64)` parse; no crate.

## Feature 1 — Least-squares projection (per-window)

Replace the first-vs-last slope with a robust least-squares fit, tracked
**per rate-limit window** rather than per account.

- **Sampling:** `record_usage_samples()` records one sample per *window* keyed
  `"{account_key}#{label}"` (e.g. `"codex:me@x#7d"`, `"gemini:me@x#3-pro"`),
  keeping ~30 min of history; a window "reset" is detected when utilization
  drops by **> 2.0** (up from 1.0) and clears that window's samples.
- **Slope:** `slope_per_second(samples) = Σ(xᵢ−x̄)(yᵢ−ȳ) / Σ(xᵢ−x̄)²`
  over `(t_seconds, utilization)`.
- **Gate:** ≥ **4** samples (up from 3), ≥ **600 s** span (up from 300 s),
  slope > `0.0005` %/s, current < 100 %.
- **Projection:** per window, `secs_to_full = (100 − util) / slope`; keep only
  if the window hits full *before* it resets; the row shows the **soonest**
  such hit across its windows: `"↗ on pace to hit limit ~3:40 PM"`.
- Helpers: `projectable_windows(key) -> [(label, util, resets_at)]`,
  `projection_text(key) -> Option<String>`, `window_name(label)` for display
  (`5h`→`5-hour`, `7d`→`weekly`, `30d`→`monthly`).

**Order rationale:** per-window sampling is a prerequisite for Gemini (which has
multiple per-model windows), so land it first.

**Tests:** `slope_per_second` on known series; gate thresholds; per-window key
formation; reset detection; "resets before full → no projection".

## Feature 3 — In-app OAuth re-login (Claude + Codex)

A coral **Login** action appears on any `needs_action` (token-rejected),
inactive, switchable row. Clicking it runs a native PKCE `authorization_code`
flow and writes fresh tokens **only** to the saved-profile file.

### Core invariant

Re-login writes fresh tokens **only** into the saved-profile snapshot
(`~/.config/pitstop/accounts/<provider>-<email>.json`). It never writes
`~/.claude/.credentials.json`, `~/.claude.json`, or `~/.codex/auth.json`. Because
the feature is scoped to inactive accounts, the re-logged-in account is by
definition not live, so a running `claude`/`codex` is structurally unaffected.

### Coordinator flow (`oauth.rs`)

1. Guard: refuse a second login while one is `login_in_flight`.
2. Generate PKCE (verifier, `challenge = base64url(SHA256(verifier))`, `state`).
3. Build the adapter's authorize URL; bind the loopback; open the browser
   (`xdg-open`).
4. Await `(code, state)` from the loopback (timeout ~90–180 s). **Paste
   fallback:** on timeout/failure, `zenity --entry` prompts for the pasted
   value (accept a full redirect URL, `CODE#STATE`, or `code=…&state=…`); if
   `zenity` is absent, `notify-send` the instructions and copy the authorize URL
   to the clipboard via `xclip`. The token-exchange `redirect_uri` must
   byte-match whichever authorize URL produced the code.
5. Verify `state`; exchange the code; verify the browser identity matches the
   row's email/account-id (else write nothing and notify the mismatch).
6. Patch the saved blob and persist to the profile file; `refresh_all()` heals
   the row (pill → normal chip) on the next cycle.

### Provider parameters (from the Mac spec; `[verify]` items validated by round-trip)

| | Claude Code | Codex |
|---|---|---|
| Authorize | `https://claude.ai/oauth/authorize` | `https://auth.openai.com/oauth/authorize` |
| Token | `https://platform.claude.com/v1/oauth/token` → fallback `https://console.anthropic.com/v1/oauth/token` | `https://auth.openai.com/oauth/token` |
| client_id | `9d1c250a-e61b-44d9-88ed-5944d1962f5e` | `app_EMoamEEZ73f0CkXaXp7hrann` |
| Scopes | `org:create_api_key user:profile user:inference user:sessions:claude_code user:mcp_servers user:file_upload` | `openid profile email offline_access api.connectors.read api.connectors.invoke` |
| PKCE | S256; verifier 32 **or** 64 bytes (don't hardcode) | S256; verifier 64 B, state 32 B |
| Extra authorize params | `code=true` (paste mode only) | `id_token_add_organizations=true`, `codex_cli_simplified_flow=true`, `originator=codex_cli_rs` |
| Loopback | `http://localhost:<port>/callback` (proven accepted; Mac E2E); paste redirect `https://platform.claude.com/oauth/code/callback` | fixed `http://localhost:1455/auth/callback`, fallback `1457` |
| Exchange | POST **JSON**, `state` in body | POST **form-urlencoded**, no `state` |
| Response | `access_token` `sk-ant-oat01-…`, `refresh_token` `sk-ant-ort01-…`, `expires_in` → `expiresAt=(now+expires_in)*1000` | `id_token`, `access_token`, `refresh_token`; expiry from `id_token.exp` |
| Identity | `GET https://api.anthropic.com/api/oauth/profile` (Bearer); degrade to confirm dialog if unavailable | decode `id_token` JWT → `email` + `https://api.openai.com/auth`→`chatgpt_account_id` |
| Persist | patch `claudeAiOauth` token fields, preserve `subscriptionType`/`rateLimitTier` → profile file | patch `tokens.{access,refresh,id}` + `last_refresh`, preserve `OPENAI_API_KEY`/`account_id`, compact/sorted → profile file |

### Wiring

- `app.rs`: `Action::Login { key }`; `perform_login(key)` spawns the coordinator
  with the right adapter; `login_in_flight` flag; on success clear backoff +
  `refresh_all()`.
- `tray.rs`: render the Login action on eligible rows.
- Reuse existing `credentials.rs` patch helpers (Claude) and add Codex
  patch/JWT-decode helpers in `codex.rs` where missing.

**Tests:** PKCE charset/length + challenge; authorize-URL assembly per adapter;
callback parsing (loopback query + 3 paste formats); blob-patch round-trips
(Claude preserves plan fields; Codex stays compact, `last_refresh`/`account_id`
correct); identity match/mismatch decisions.

## Feature 4 — Gemini provider (Antigravity surface)

A merged **"Gemini"** row per Google account showing live Code Assist usage,
switchable, auto-switch-eligible, with a Login safety net.

### Credential storage (Linux)

- **Live:** GNOME keyring, go-keyring item `service=gemini, account=antigravity`,
  value `"go-keyring-base64:" + base64(JSON)` where JSON =
  `{"token":{"access_token","token_type":"Bearer","refresh_token","expiry":ISO8601},"auth_method":"consumer"}`.
  Treat the whole `go-keyring-base64:` string **opaquely** for snapshot/restore;
  decode only to read tokens for polling/refresh.
- **Snapshots:** `~/.config/pitstop/accounts/gemini-<sanitized-email>.json`
  (`0600`), preserving the opaque keyring value; non-secret metadata (email,
  plan, savedAt) in `~/.config/pitstop/gemini-profiles.json`.

### Identity (Linux-specific)

The Antigravity blob carries **no email** and `google_accounts.json` is absent.
Resolve the account email via a **Google `userinfo`** call with the access token
(`GET https://openidconnect.googleapis.com/v1/userinfo`, Bearer) → `email`.
Cache per keyring blob (in-memory). Key = `"gemini:<email>"`.

### Usage (`gemini.rs`)

1. Decode the keyring blob; if `expiry` is past, refresh in memory (refresh-token
   grant, form-urlencoded, `https://oauth2.googleapis.com/token`, Antigravity
   client `1071006060591-…` / secret `GOCSPX-…` **[verify]**).
2. `POST https://cloudcode-pa.googleapis.com/v1internal:loadCodeAssist`
   body `{"metadata":{"ideType":"IDE_UNSPECIFIED","platform":"PLATFORM_UNSPECIFIED","pluginType":"GEMINI"}}`
   (Mac sent `DARWIN_ARM64`; use a neutral/`LINUX_X86_64` value — **[verify]** the
   field doesn't gate the response) → cache `cloudaicompanionProject`; derive the
   **plan chip** from `paidTier.name` (`"Google One AI Pro"`→`"AI Pro"`) or
   `currentTier.name` (`"Gemini Code Assist"`→`"Code Assist"`).
3. `POST …:retrieveUserQuota` body `{"project":"<id>"}` → `buckets`.
4. Map each bucket → window: `label` = shortened `modelId`
   (`gemini-3.1-pro-preview` → `3.1-pro`: drop `gemini-` prefix + trailing
   `-preview`), `utilization = (1 − remainingFraction) × 100`,
   `resets_at = resetTime` (RFC3339). Skip buckets missing `remainingFraction`.
5. **Binding** = highest-utilization bucket → row's main bar + menu-bar pool.
   **Extras** = next up-to-2 non-zero models (`3-pro 22% · 2.5-flash 5%`).
6. No project / empty buckets → identity/presence row, no bar (neutral).
7. Errors reuse `ApiError`: 401/403 → `Unauthorized` (→ Login pill), 429 →
   `RateLimited` (backoff). Plugs into existing status/backoff/`needs_action`.

### Switching (`gemini_store.rs`)

- **Snapshot on every refresh** (`capture_current`): write the live keyring blob
  to the profile file (skip if byte-identical); persist metadata.
- **`switch_to(email)`:** snapshot current first (never strand an outgoing
  refresh token), then write the target's saved blob back into the keyring
  in-place (preserving the `go-keyring-base64:` prefix).
- **Inactive-token refresh:** refresh stale *saved* snapshots via the Google
  refresh grant and store rotated tokens back; **never** rewrite the live keyring
  just to poll (Codex model).
- **Notification after switch:** *"Switched Gemini to <email> — quit & reopen
  Antigravity to pick it up."* + the **ToS rotation caveat**.

### Auto-switch + menu bar

- Gemini joins `evaluate_auto_switch` as a third provider (same threshold +
  cooldown, opt-in). `perform_gemini_switch` swaps the keyring blob.
- Gemini usage joins the `most_urgent` menu-bar pool; the default
  `active_claude_code` menu-bar source is unchanged.

### Re-login (Login pill, safety net)

Reuse the coordinator. Google installed-app clients accept an arbitrary loopback
port → fully automatic, no paste. `GeminiLoginAdapter`: client `1071006060591-…`;
scopes `cloud-platform userinfo.email userinfo.profile cclog experimentsandconfigs`;
authorize `https://accounts.google.com/o/oauth2/v2/auth`; loopback redirect
`http://127.0.0.1:<port>/oauth2callback`; token exchange form-urlencoded at
`https://oauth2.googleapis.com/token`; PKCE S256; identity from `userinfo` (or
`id_token`); persist to the keyring + snapshot. Google refresh tokens rarely die,
so this is a safety net, not a common path.

### Spike gate (first task of this feature)

Before building anything else in Feature 4, prove we can **read** the Antigravity
keyring item from Rust via `secret-service` and successfully call `loadCodeAssist`
+ `retrieveUserQuota` with the decoded token, and resolve the email via
`userinfo`. If the keyring item can't be read (locked/absent/schema mismatch) or
the endpoints reject the token, **pause Feature 4** and ship the rest; report
findings. This de-risks the largest unknown up front.

**Tests:** keyring `go-keyring-base64:` decode/encode round-trip; model-name
shortening; `retrieveUserQuota` parse → windows + binding + extras (empty/partial
→ no bar); `loadCodeAssist` parse → project id + plan chip; switch blob-building
(prefix preserved); Google refresh request builder; authorize-URL builder.

## Feature 5 — Updater (rebuild & relaunch)

- **Version:** `env!("CARGO_PKG_VERSION")`; bump `Cargo.toml` to `0.3.1` for
  parity with the Mac feature set (documented in `CHANGELOG`).
- **Check:** daily (86 400 s throttle; `last_update_check` timestamp) GET the
  GitHub Releases API for the **Linux** repo (`Livin21/pitstop-linux`), take the
  latest non-prerelease tag, semver-compare to the running version.
- **Menu:** always show the current version (disabled line); when an update
  exists show **`Update & Relaunch`** (and the release notes URL).
- **Rebuild (`can_rebuild`):** locate the source checkout via a path recorded by
  `install.sh` in `~/.config/pitstop/repo_path` (written at install time). Then
  `git -C <repo> pull --ff-only` → `cargo build --release` → copy the binary to
  `~/.local/bin/pitstop` → **fork+exec** the new binary and exit the old process
  (tray reconnects via `ksni`). If the checkout is missing/dirty or the build
  fails → notify + open the releases page instead (no partial install).
- `install.sh` change: after building, write the absolute repo path to
  `~/.config/pitstop/repo_path`.

**Tests:** semver parse/compare (incl. `v` prefix, pre-release skip); "up to
date → no menu item"; checkout-path resolution + missing-path fallback decision.

## Build order (independently verifiable phases)

0. **Scaffold** — add deps; create empty modules; `mod` declarations; bump version.
1. **Projection** (Feature 1) — per-window least-squares. Foundational.
2. **Dashboard links** (Feature 2) — quick win.
3. **OAuth re-login** (Feature 3) — `oauth.rs` + `loopback.rs` + adapters + Login
   pill + wiring. Verify by healing an expired Claude row (and Codex).
4. **Gemini/Antigravity** (Feature 4) — **spike first**, then usage → switch →
   menu → auto-switch → Login. Highest risk.
5. **Updater** (Feature 5) — `updater.rs` + `install.sh` + menu.

Each phase: builds clean, unit tests pass, `cargo clippy` clean, manually
verifiable, one commit.

## Risks / to-verify during implementation

1. **[Gemini] keyring read** gates all of Feature 4 — spike is its first task.
2. **[Gemini] go-keyring Secret Service schema** must match for read *and*
   write-back (attributes `service`/`username`, label, base64 value + prefix).
3. **[Gemini] identity via `userinfo`** — confirm endpoint + that the Antigravity
   token's scopes return `email`.
4. **[Gemini] Antigravity client_secret** is reverse-engineered (`[verify]`);
   handle refresh failure by degrading to observe/identity-only for that account.
5. **[Gemini] `loadCodeAssist` platform metadata** on Linux — verify a neutral
   value is accepted.
6. **[Claude] token-endpoint host** (`platform.claude.com` vs
   `console.anthropic.com`) — try primary, fall back; `redirect_uri` must
   byte-match authorize.
7. **[Codex] loopback port 1455/1457 contention** with a running `codex login`.
8. **[Updater] re-exec** must not orphan the tray; rebuild must be atomic
   (build to a temp path, then move); dirty checkout → abort to release-page.
9. **[secret-service] async client** must cooperate with the single tokio task
   (use the tokio runtime feature; avoid blocking the select loop).

## Testing strategy

- **Unit (no network/keyring):** PKCE, authorize URLs, callback/paste parsing,
  blob-patch round-trips (Claude/Codex/Gemini), model-name shortening, quota
  parse, least-squares slope, semver compare, dashboard URLs.
- **Manual E2E (documented per phase):** Claude & Codex re-login heal an expired
  row without touching the live store; Gemini usage renders and a switch
  round-trips (both accounts' saved blobs restored correctly); updater detects a
  release and rebuild+relaunch works from the recorded checkout.

## Global constraints

- Rust 2021; keep the single-`tokio`-task model; no new threads/locks in the
  render path.
- Secrets stay in `0600` files or the GNOME keyring; never logged.
- Follow existing module/style conventions; don't over-split (match the app's
  flat `src/` granularity).
- Update `README.md` (new provider, re-login, updater, ToS caveat) and add a
  `CHANGELOG.md` mirroring the Mac entries as features land.
