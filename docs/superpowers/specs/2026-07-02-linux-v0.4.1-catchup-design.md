# PitStop-Linux → macOS v0.4.1 Feature Catch-Up — Design

**Date:** 2026-07-02
**Status:** Approved design, ready for implementation planning
**Component:** PitStop for Linux (Rust, `ksni` system-tray app)
**Upstream:** ports macOS PitStop v0.3.1 → v0.4.1 (`../pitstop`, range `3feafd3..408b7be`)

## Problem

The Mac app shipped v0.4.0/v0.4.1: one new feature (per-model **scoped weekly
limits** — "Fable" — from the usage API's new `limits[]` array) plus a batch of
robustness and status-polish fixes. A delta triage against the current Linux
code classified every upstream change as applies / already-handled / macOS-only.
This design covers the applies/partial set. The Linux app version moves to
**0.4.1**.

## Out of scope (triaged)

- **macOS-only:** Keychain `#staging` recovery; `security`-CLI hex corruption;
  `google_accounts.json` active-email + CLI-blob-ownership fixes (no Gemini CLI
  on Linux); paste-window over-release; `arc4random` fallback; notification
  banners while active app; SMAppService toggle sync; bar dimming via alpha;
  tracking-area retain cycle.
- **Already handled by the Linux architecture:** refresh-queueing (serial
  single-task engine); Gemini project-cache invalidation (no cache — fetched
  per cycle); fractional-second RFC3339 (chrono accepts natively); subprocess
  pipe deadlock (tokio `.output()` drains both); non-JSON-200 single-use-code
  replay (Rust `?` propagates `Malformed` out of the host loop); loopback
  stray-connection resilience (accept loop already skips non-callback
  requests); launch-at-login drift (checkbox reads the autostart file live).
- **Deliberately skipped:** bar-label column alignment (plain-text tray;
  purely cosmetic).

## Plan 1 — Fable scoped weekly limits (feature)

Upstream spec: `../pitstop/docs/superpowers/specs/2026-07-02-fable-scoped-limits-design.md`.
The `api.anthropic.com/api/oauth/usage` response gained a generic `limits`
array; scoped entries look like:

```json
{ "kind": "weekly_scoped", "group": "weekly", "percent": 13,
  "resets_at": "2026-07-05T00:00:00Z",
  "scope": { "model": { "display_name": "Fable" } } }
```

`seven_day_opus` / `seven_day_sonnet` are permanently `null` — dead plumbing.

### Changes (`usage_api.rs`, `model.rs`, `app.rs`)

1. **Data model:** `pub struct ScopedWindow { pub label: String, pub window: UsageWindow }`;
   `UsageReport` gains `pub scoped: Vec<ScopedWindow>`; **remove**
   `seven_day_opus` + `seven_day_sonnet` fields and their parse lines.
2. **Parse:** a `limit_window_entry(&Value) -> Option<UsageWindow>` helper reads
   `percent` (f64; note: NOT `utilization`) + `resets_at`. `parse()` extracts
   `limits[]`; scoped = entries with `kind == "weekly_scoped"`, label from
   `scope.model.display_name` (fallback `"Scoped"`). Unknown `kind` ignored.
   Legacy 5h/7d still come from the top-level fields; if absent, fall back to
   the `limits` entry with `kind == "session"` / `"weekly_all"`.
3. **Binding:** `max_utilization()` chain-maxes over scoped windows;
   `binding_window()` picks the max across `[5h, 7d] + scoped`;
   `IndicatorMetric::Binding`'s has-data guard includes scoped. This makes the
   menu-bar %, most-urgent pool, auto-switch, and threshold notifications react.
4. **Rows + projection:** `build_row()` Claude branch pushes one `window_line`
   per scoped window after 7d; the Opus/Sonnet extras arms are removed (keep
   only "Extra X%"); `record_usage_samples()` records scoped windows under
   `"{key}#{label}"`; `projectable_windows()` returns them; `status_tip()`
   appends `" · {label} {percent}"` per scoped window. `window_name()` already
   passes unknown labels through.
5. **Projection floor** (v0.4.1, d062687): in `projected_full_from_samples`,
   after the `secs_to_full <= 0` check, suppress the projection when
   `current < 25.0 && secs_to_full > 10800.0` (3 h) — a barely-used window
   can't cry wolf.

### Tests

Parse: scoped entry from a real-shaped payload (label/percent/reset); label
fallback when `display_name` missing; legacy fallback via `session`/`weekly_all`;
unknown `kind` tolerated; `max_utilization`/`binding_window` include scoped;
IndicatorMetric has-data guard. Projection floor: 2% used + ETA 9h → None;
26% used same slope → Some; 2% used + ETA 2h → Some.

## Plan 2 — Robustness fixes + status polish (batch)

### 2a. Single-instance lock (`main.rs`) — highest priority (bug reproduced live)
Before `run_tray()`: open/create `~/.config/pitstop/pitstop.lock`, take
`flock(LOCK_EX | LOCK_NB)` (via `libc` or `rustix`/`nix` — prefer a plain
`libc::flock` call to avoid a new heavyweight dep; `libc` is already in the
tree transitively), and on failure print "another PitStop instance is running"
and exit(0). Keep the fd open for process lifetime (leak it deliberately).
One-shot flags (`--check`, `--gemini-spike`, `--export-icon`) do NOT take the
lock — they don't touch live credential files in a conflicting way and must
work alongside a running tray.

### 2b. Gemini re-login PATCHES the saved blob (`oauth.rs`) — real credential bug
`GeminiLoginAdapter::persist` currently calls `build_antigravity_blob` from
scratch; Google omits `refresh_token` on re-consent, so the rebuilt blob can
lose it → profile permanently unrefreshable. Fix: read the existing saved
snapshot (`secret_store::read`), and if present, `gemini::patch_antigravity_blob`
it (which preserves the old `refresh_token` when the fresh tokens lack one and
preserves the stored wrapper form); build from scratch only when no snapshot
exists. Extend `patch_antigravity_blob` if needed so a `Some(refresh)` fresh
token updates the field while `None` preserves.

### 2c. Claude switch rollback on half-failure (`claude_store.rs`)
`switch_to` writes `~/.claude/.credentials.json` then `set_oauth_account`
(`~/.claude.json`). Read the previous live blob first; if `set_oauth_account`
fails, restore the previous blob so the two files can't disagree, then
propagate the error.

### 2d. Preserve API-key-only Codex auth across switches (`codex.rs`/`codex_store.rs`)
`codex::credentials()` returns `None` for an API-key-only `auth.json` (no
`tokens.access_token`), so `capture_current` can't snapshot it and a switch
would clobber it. Mirror the Mac fix: when writing a saved blob into the live
file during `switch_to`, merge any `OPENAI_API_KEY` present in the *current
live* file into the blob being written (a `preserving_api_key(live, saved)`
helper with unit tests: live has key + saved null → key kept; both have →
saved wins per Mac semantics — match the Mac diff exactly).

### 2e. `write_atomic` preserves symlinks (`util.rs`)
`rename(2)` replaces a symlink node with a regular file. Fix: canonicalize the
destination (`std::fs::canonicalize` on the path if it exists — resolving to
the symlink target) before the tmp+rename, so dotfiles-managed credential
files stay symlinks. Unit test with a symlinked target in a temp dir.

### 2f. Loopback: Deny = cancel (`loopback.rs` + `oauth.rs`)
A callback request carrying `error=access_denied` (no code) currently gets
skipped by the accept loop (waits out the timeout). Parse `error` in the
callback query; yield a distinct outcome so `run_login` fails fast with
"Sign-in was cancelled" instead of a spurious timeout.

### 2g. External CLI re-login heals `needs_action` in one refresh (`app.rs` + stores)
`capture_current` (Claude + Codex) returns whether the live credentials
CHANGED since the last snapshot; when they changed, clear that account's
`needs_action`/backoff/failure_count (`clear_fetch_error`) so a `claude` /
`codex login` performed outside PitStop heals the row on the next 2-min cycle
instead of after the 1-h backoff. (Gemini: same via the per-cycle snapshot if
cheap; otherwise note as N/A since Google tokens rarely die.)

### 2h. Prune orphaned per-account state (`app.rs`)
`Action::Remove` (and account-disappearance paths) currently clear usage,
`fetch_error`, `next_fetch_allowed`, `failure_count` — but leak `needs_action`,
`notified_bucket`, and `usage_history` (`"{key}#…"` entries). Clear all three
so ghost accounts can't drive the most-urgent menu-bar reading or projections.

### 2i. Countdown fixes (`format.rs`)
`relative_short`: <60 s renders `0m` → `<1m`. `relative`: elapsed dates render
`in 0s` → `now`. Unit tests for both boundaries.

### 2j. Status polish (`app.rs` row_status + Codex row)
- **Age-gated transient errors:** if the key is in `needs_action`, OR there is
  no cached data, OR the data is older than 600 s → keep
  `⚠︎ {error} · showing {time} data`; otherwise render a muted info line
  (error text only, no ⚠︎, no "showing" caveat).
- Stale timestamps use `format::short_clock` (no seconds).
- The live Codex row waiting on a fresh token: replace the token-mechanics
  sentence with `Last seen {short_clock}` (with data) / `No usage data yet`
  (without); keep the existing bars.
- **Dangling "Extra –":** only push the "Extra X%" line when
  `extra_usage_utilization` is `Some`.

### 2k. Rounded-percent color (`app.rs dot()`)
`dot()` thresholds on raw f64 while the text shows the rounded percent (89.6 →
"90%" but yellow). Round first, threshold on the rounded value (≥90 red, ≥70/75
per current thresholds — keep existing threshold values, just round the input).

### 2l. Auto-switch label discloses Gemini (`tray.rs`)
Settings item text: "Auto-switch when an account runs low" →
"Auto-switch when low (Claude, Codex, Gemini)" (or closest ksni-friendly
wording per the Mac copy).

### Version + docs
`Cargo.toml` → 0.4.1 (+ lock sync). CHANGELOG entries mirroring the Mac's
0.4.0/0.4.1 sections (Linux-applicable subset). README: mention scoped weekly
limits; note single-instance behavior.

## Build order

Plan 1 (Fable) and Plan 2 (fixes) are independent; execute Plan 1 first (it
touches `usage_api.rs`/`app.rs` structurally — removing opus/sonnet — and Plan
2j's extras guard lands cleaner after the extras block shrinks). Within Plan 2,
2a (lock) first.

## Global constraints

Unchanged from v0.3.1: Rust 2021; single-tokio-task engine; secrets only in
0600 files or the GNOME keyring, never logged; secret-bearing structs don't
derive Debug; reuse `ApiError`; every task ends green (build/test/clippy) with
one commit. Re-login writes only saved-profile snapshots; live stores are
mutated only by an explicit switch (documented exception: same-account Claude
active-token refresh).

## Risks

1. **[Fable] `percent` field vs `utilization`** — the limits entries use
   `percent`; don't reuse the legacy field name when parsing.
2. **[2d] Mac merge semantics** — replicate `preservingAPIKey`'s exact
   precedence from the Mac diff, don't guess.
3. **[2e] canonicalize on a non-existent path errors** — only canonicalize
   when the file exists; else write directly.
4. **[2a] lock file location** — must be a path that exists early
   (`~/.config/pitstop` is created by settings/store init; create-if-missing
   before locking).
5. **[2b] patch must preserve wrapper form + refresh_token** — extend the
   existing form-preserving `patch_antigravity_blob`, don't fork a second
   patcher.
